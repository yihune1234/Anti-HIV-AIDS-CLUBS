## Role Definitions

### Public
- Unauthenticated visitors
- Can view public pages (Home, About, Vision, Awareness)
- Can ask anonymous questions
- Can register for an account
- Can contact via email form

### Member
- Authenticated users with basic membership
- All public access +
- View and interact with stories, resources, gallery, events
- Like and comment on content
- Register for events and sessions
- Manage own profile

### Admin
- Full system access
- All member access +
- Create, edit, delete all content
- Manage users and roles
- Approve/reject content submissions
- Configure system settings
- Generate reports and analytics
- Mark attendance for events/sessions

## Security Notes

1. All admin routes are protected by `AdminRoute` guard
2. All member routes are protected by `ProtectedRoute` guard
3. JWT tokens are used for authentication
4. Passwords are hashed using bcrypt
5. Role-based access control (RBAC) implemented
6. Session timeout configurable by admin
7. Max login attempts configurable by admin
8. Email verification optional (configurable)
9. Content approval workflow prevents unauthorized publishing
10. All API endpoints validate user roles before processing requests
